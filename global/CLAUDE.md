# グローバルルール（全プロジェクト共通）

## メタデータ
- macOS, Apple Silicon

## 言語・応答スタイル
- 日本語で応答すること
- コードのコメントや変数名も日本語を使用する
- 変更点の要約を1-2行で簡潔に添える
- 不要な前置きや繰り返しを避ける

## コード変更
- コードの変更は最小限にとどめる
- 既存コードのフォーマット・リンター設定に従う
- 新規ファイル作成は確認不要。ただし破壊的変更の場合は事前に日本語で確認する
- エラー発生時は確認なしに自動修正を試みる
- READMEやドキュメントはプロジェクトに必要と判断したら積極的に作成する

## Git
- コミットメッセージ: プレフィックスは英語(feat:, fix:, chore:等)、本文は日本語
- .envファイルは絶対にコミットしない。.gitignoreに含まれていなければ追加する
- .envファイルの内容は読み取らない
- 代わりに.env.exampleをダミー値(dummyToken等)で作成し、git管理する

## セキュリティ
- APIキーやシークレットをコードにハードコードしない
- .envファイルを絶対にgitにコミットしない
- .envファイルの中身をReadツールで読み取らない
